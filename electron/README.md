# iRacing Career Manager — Electron Build Guide

Everything you need to go from source → distributable Windows `.exe`.

---

## Prerequisites

| Tool | Why | Install |
|------|-----|---------|
| Node.js 18+ | Runs Electron + build tools | https://nodejs.org |
| Python 3.10+ | Compiles bridge.py | https://python.org |
| PyInstaller | Bundles Python → .exe | `pip install pyinstaller` |

---

## Project Structure (after setup)

```
your-project/
├── index.html              ← your existing app
├── src/                    ← your existing src/
├── bridge.py               ← your existing bridge script
├── fonts/                  ← generated by download_fonts.js
│   ├── fonts.css
│   └── *.woff2
├── bridge/
│   └── bridge.exe          ← generated by build_bridge.bat
├── electron/
│   ├── main.js             ← Electron main process
│   ├── preload.js          ← contextBridge API
│   ├── package.json        ← npm config + electron-builder
│   ├── build_bridge.bat    ← compiles bridge.py
│   └── download_fonts.js   ← downloads Google Fonts locally
└── build-assets/
    ├── icon.ico            ← app icon (you supply this)
    └── LICENSE.txt         ← shown in installer
```

---

## Step 1 — Copy files into your project

Place all files from this package into your project root as shown above.

---

## Step 2 — Install Node dependencies

```bash
cd electron
npm install
```

---

## Step 3 — Bundle Google Fonts (offline)

```bash
cd electron
node download_fonts.js
```

Then open `index.html` and replace:

```html
<!-- REMOVE this (or any Google Fonts link tag) -->
<link href="https://fonts.googleapis.com/css2?family=Inter..." rel="stylesheet">

<!-- ADD this instead -->
<link rel="stylesheet" href="fonts/fonts.css">
```

> **Why?** Electron apps can't reliably hit Google's CDN. Bundle the fonts so
> the app looks correct 100% of the time, even offline.

---

## Step 4 — Build bridge.exe from bridge.py

```bash
cd ..         ← back to project root (where bridge.py lives)
electron\build_bridge.bat
```

This runs PyInstaller and outputs `bridge/bridge.exe`.  
If your bridge.py has extra dependencies, install them first:

```bash
pip install irsdk flask flask-cors
```

> **What bridge.exe does:** It's your existing `bridge.py` compiled to a
> standalone Windows executable. Electron spawns it as a background child
> process on startup — users never see a terminal window.

---

## Step 5 — Add an app icon

Create `build-assets/icon.ico` — a 256×256 (or multi-resolution) `.ico` file.
You can convert any PNG at https://convertio.co/png-ico/

Also create `build-assets/LICENSE.txt` with your license text (shown in the
NSIS installer). A one-liner is fine:

```
Copyright (c) 2025. All rights reserved.
```

---

## Step 6 — Build the Windows installer

```bash
cd electron
npm run build:win
```

This produces two files in `electron/dist/`:

| File | Description |
|------|-------------|
| `iRacingCareerManager-setup-1.0.0.exe` | NSIS installer (recommended for distribution) |
| `iRacingCareerManager-portable-1.0.0.exe` | Single-file portable, no install needed |

---

## Development / Testing

Run the app locally without building:

```bash
cd electron
npm start
```

The app opens in a window. DevTools opens automatically in dev mode.
The bridge won't auto-start until `bridge/bridge.exe` exists.

---

## How the bridge integration works

1. `main.js` spawns `bridge.exe` as a hidden background process on launch.
2. The bridge listens on `http://localhost:54321` (unchanged from your existing setup).
3. `preload.js` exposes `window.electronBridge` to the renderer — this is what
   your existing `bridge.js` already checks for:
   ```js
   if (typeof window !== 'undefined' && window.electronBridge) { ... }
   ```
4. Users can toggle bridge auto-start from the iRacing Bridge panel in Settings.
5. If the bridge crashes, `main.js` auto-restarts it after 3 seconds.
6. The bridge process is killed cleanly when the app quits.

## Distribution

The NSIS installer:
- Lets users choose install directory
- Creates a Desktop + Start Menu shortcut
- Includes an uninstaller

The portable `.exe` is a single file — no install, runs from anywhere.
Good for users who want to try it before committing.

## Troubleshooting

**Bridge not starting:**
Check that `bridge/bridge.exe` exists. Run `build_bridge.bat` if not.

**Fonts look wrong / missing:**
Run `download_fonts.js` and verify `index.html` points to `fonts/fonts.css`.

**App shows blank screen:**
Open DevTools (`Ctrl+Shift+I` in the app window) and check the console for
path errors. Most likely `index.html` references `src/` files with paths
that break inside the packaged `resources/app/` directory — all relative
paths like `src/ui/core.js` should work fine as long as you haven't used
absolute paths anywhere.

**PyInstaller fails:**
Make sure all `bridge.py` imports are installed:
```bash
pip install irsdk flask flask-cors pyinstaller
```
